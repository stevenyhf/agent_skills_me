# Roundtable Skill — 多人格圆桌对话系统

## 核心定位

选定一个话题 + 3-8 位人格（从已安装的 `*-method` / `dialogue-*` Skills 中选择），自动完成四阶段流程：**并行观点采集 → Dialogue Master 主持碰撞 → 综合分析报告 → 保存到 `~/文档/`**。

## 四阶段执行流程

### Phase 0：意图解析 + 人格匹配（编排层，主会话内执行，约1-2分钟）

1. **解析**用户输入的**话题**和**人选**（3-8人）
2. 若用户**未指定人选**，根据话题性质自动推荐：
   - AI/科技话题 → 马斯克/黄仁勋/张一鸣/梁文锋/王兴
   - 商业/创业话题 → 贝索斯/马云/刘强东/雷军
   - 哲学/人文话题 → 孔子/老子/苏格拉底/柏拉图
   - 跨界话题 → 推荐古今混搭（古代哲人 + 现代企业家）
3. 检查 `~/.agents/skills/` 中对应 Skill 是否已安装，未安装则提示用户安装命令
4. 向用户**快速确认**（1行，不啰嗦）
5. 可选：执行 `web_search` 拉取话题最新动态（30秒，失败不阻塞）

### Phase 1：并行观点采集（Sub-agents，每人独立，约30-60秒）

对每位人选执行 `sessions_spawn(mode="run")`：

```
你现在扮演 **{人物名}({人物 Skill 名})**，使用以下方法论框架思考问题。

## 你的核心思维框架
{从对应 Skill 的 SKILL.md 中提取：3-5个核心框架摘要 + 人物代表性引语，共300-500字}

## 话题
{话题描述 + 用户原始问题}

## 背景参考（如果有）
{web_search 结果摘要，不超过300字}

## 输出格式
## {人物名}对「{话题}」的观点

### {框架A}判断（150-200字）
### {框架B}判断（150-200字）
### {框架C}判断（150-200字）
### 我的核心判断（150-200字）
### 最危险/最值得关注的地方（100字）
### 金句
（用这个人格的风格说一句总结性的话）

字数：800-1500字，风格要像{人物名}本人——使用该人物最典型的表达方式和思维特征。
```

所有人物并行 spawn 后执行 `sessions_yield`，等待全部完成。

**关键设计点**：
- 每位子代理**独立思考**，不知晓其他人格的存在（避免从众）
- 编排层必须从 Skill 文件中**提取框架摘要注入 prompt**，保证人格一致性
- 若子代理超时（> 3分钟）跳过并说明

### Phase 2：Dialogue Master 主持碰撞（主会话串行，约3-5分钟）

**Step 2.1：观点碰撞矩阵**（表格）

| 焦点维度 | 人物A | 人物B | 人物C | 人物D |
|----------|-------|-------|-------|-------|
| 时间/核心判断 | ... | ... | ... | ... |
| 核心瓶颈/路径 | ... | ... | ... | ... |
| 最大风险/警告 | ... | ... | ... | ... |
| 哲学观/思维特征 | ... | ... | ... | ... |

**Step 2.2：3段虚拟对话碰撞**
基于 Dialogue Master 的查理·罗斯式精准追问 + 圆桌派氛围营造：
- **交锋1**：定义层面的最大分歧
- **交锋2**：路径层面的核心矛盾
- **交锋3**：赢家/利益层面的最尖锐分歧
- 每段对话：每人 80-150字回应，共320-600字/段
- 必须包含至少1次**直接反驳** + 查理式底层逻辑追问

**Step 2.3：破局提问**（Dialogue Master 僵局破局策略）
问所有嘉宾："如果 X 年后回头看今天的判断，你最可能'错'在哪里？"
每人 80-100字的自我反省式回应。

### Phase 3：综合分析报告（主会话，researcher-yhf 框架，约2-3分钟）

报告结构（严格按此顺序）：

```markdown
# 圆桌对话报告：{话题}

**日期**：{YYYY-MM-DD}
**参与方**：{人物} × {人数}
**主持人**：Dialogue Master
**报告生成**：researcher-yhf
**执行机制**：Roundtable Agent Protocol v1.0

## 执行摘要
500字以内，核心结论 + 最大共识/分歧一句话总结

## 一、各位嘉宾独立观点摘要（每人一节）
- 核心论断（1句）
- 关键推理链（3-4条bullet）
- 最大警告/洞察（1-2句）
- 金句（1句）

## 二、观点碰撞矩阵（表格）
4-6个维度 × N位人物

## 三、Dialogue Master碰撞对话（3段虚拟圆桌对话 + 破局提问）

## 四、共识与分歧分析
- 共识点（N位一致，bullet list）
- 主要分歧（带倾向方说明的表格）
- 最有价值的洞察（主持人精选3-5条）

## 五、对用户决策建议（分层：创业者/管理者/投资人/个人）

## 六、金句合辑（每人一句，加粗斜体）
```

### Phase 4：本地保存 + 展示（约10秒）

**保存路径**：`~/文档/圆桌对话-{话题关键词}-{日期}.md`

保存后向用户展示：
- 报告摘要（3-5个核心发现）
- 金句合辑
- 文件路径

## 执行规则

### 规则1：人格一致性优先
子代理 task prompt 必须包含该人物 Skill 的核心框架摘要（≥3个框架 + 核心引语），不可泛泛而谈。

### 规则2：独立性保护
Phase 1 的所有子代理并行运行，互不知晓彼此。Phase 2 的碰撞由主持人在主会话内基于已采集成果进行。

### 规则3：碰撞必须有张力
虚拟对话不是"总结发言"，必须包含：至少1次直接反驳 + 查理式底层逻辑追问 + 破局提问。

### 规则4：报告有结构
Phase 3 报告必须包含：执行摘要、独立观点、碰撞矩阵、虚拟对话、共识与分歧、决策建议、金句；缺一不可。

### 规则5：容错机制
- 子代理超时 >3 分钟：跳过并说明
- Skill 未安装：跳过并提示 `openclaw skills install <skill-name>` 或手动安装
- web_search 失败：不阻塞，背景材料标为"无"

## 进阶模式（可选触发）

### 模式A：追问模式
用户对某嘉宾观点特别感兴趣时：
```
# 触发词
"请 {人物X} 深入展开你的 {某观点}"
```
编排层针对该观点，再次 spawn 该人格的子代理，要求更深入的分析。

### 模式B：多轮真实迭代（高级模式，token 消耗2-3倍）
```
# 触发词
"圆桌对话（高级模式）：{话题}，请 {人物} 讨论"
```
1. Phase 1：并行采集原始观点
2. 追加轮次1：把B的观点喂给A，要求A反驳
3. 追加轮次2：把C的观点喂给B和A，要求各自回应
4. 以此类推，形成真实迭代
5. 所有轮次汇总给 Dialogue Master + researcher-yhf

注意：高级模式耗时20-30分钟，token消耗可达200K+。

### 模式C：时空穿越模式
```
# 触发词
"时空穿越对话：{话题}，请 {古代人物} 和 {现代人物} 讨论"
```
- 古代哲人（dialogue-* Skills）+ 现代企业家（*-method Skills）碰撞
- 如："请孔子、老子、苏格拉底、马斯克、维塔利克讨论 AI 伦理"
- 制造跨时空的思想对话，极具观赏性
