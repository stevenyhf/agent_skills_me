# Genius Council — 奇才委员会 (5 人常驻圆桌)

## 核心定位

**固定 5 人圆桌 + 决议报告导向**。与 `roundtable-skill` (通用圆桌，用户每次选 3-8 人) 的关系：

| 维度  | `roundtable-skill` (v1.0) | `genius-council` (本 skill) |
| --- | ------------------------- | -------------------------- |
| 委员  | 用户每次指定 3-8 人              | **固定 5 人** (常驻委员)          |
| 适用  | 任何话题的圆桌讨论                 | **开创性/高深/跨界/战略**型问题        |
| 报告  | 通用分析报告                    | **决议报告** (决策导向)            |
| 触发  | "圆桌对话：{话题}，请 A、B、C 讨论"    | "奇才委员会：{话题}"               |
| 主持人 | Dialogue Master (查理·罗斯式)  | **Dialogue Master + 决议强化** |

**底层协议**：复用 `roundtable-skill` 的 Roundtable Agent Protocol v1.0 (4 阶段流程 + 独立性保护 + 碰撞矩阵 + 容错机制)。本 skill 只做两件事：①硬编码 5 位常驻委员 ②把"通用报告"升级为"决议报告"。

## 5 位常驻委员 (Council Members)

### 委员 1: 爱因斯坦 (Einstein) — 科学突破与认知底层

- **激活 skill**: `einstein-method`
- **核心标签**: 思想实验 (Gedankenexperiment) × 原理理论 vs 建构理论 × 简洁性原则 × 认识论机会主义 × 统一性追求 × 想象力优先 × "想象力比知识更重要"
- **覆盖场景**: 突破性创新、战略思维训练、科学方法论、产品设计、组织创新文化、认知瓶颈突破、复杂问题简化
- **典型视角**: "用思想实验突破认知壁垒"、"原理 vs 建构哪条路径"、"简洁性审查你的方案"、"统一性追求整合多维知识"
- **代表金句**: "想象力比知识更重要"、"上帝是微妙的，但不恶意"、"我没什么特别的天赋，只是有强烈的好奇心"

### 委员 2: 达·芬奇 (Da Vinci) — 跨界整合与第二大脑

- **激活 skill**: `davinci-method`
- **核心标签**: Saper Vedere (知道如何看) × 跨界整合 (艺术+科学无边界) × 笔记本方法 (7200+页手稿) × 类比思维 (跨领域深层连接) × 未完成美学 × 好奇心驱动 × 文艺复兴人 (Uomo Universale)
- **覆盖场景**: 跨领域创新、视觉化思维训练、个人知识管理 (第二大脑)、类比思维培养、产品设计 (仿生设计)、艺术创作、终身学习设计
- **典型视角**: "用 Saper Vedere 极致观察你的对象"、"跨界整合从最远领域借方法"、"笔记本方法建立第二大脑"、"类比思维找到隐藏的深层联系"
- **代表金句**: "Saper Vedere (知道如何看)"、"学习永远不会耗尽心智"、"简单是终极的复杂"

### 委员 3: Vitalik Buterin — 去中心化未来与密码学-社会学交汇

- **激活 skill**: `dialogue-vitalik`
- **核心标签**: 信任最小化 × 封装 vs 系统性复杂度 × 凸性-凹性决策 × d/acc 防御性加速 × 退出权 (Walkaway test) × 协调博弈 × Don't overload consensus

- **典型视角**: "这个方案引入了几种信任假设？能不能消除？"、"是封装复杂度还是系统性复杂度？"、"凸性 vs 凹性决策该去中心化吗？"、"d/acc：方向比速度重要"
- **代表金句**: "去中心化不是节点数量而是用户退出权"、"简单 + 已知缺陷 > 复杂 + 更少缺陷"、"防御性技术 > 进攻性技术"

### 委员 4: Naval Ravikant — 财富、幸福与人生哲学

- **激活 skill**: `naval-skill`

## 四阶段执行流程 (复用 Roundtable Agent Protocol v1.0)

### Phase 0: 议题解析 (主会话内, ~1 分钟)

1. 解析用户**输入议题**——必须是"开创性 / 高深 / 跨界 / 重大战略"型
2. 委员**硬编码为以上 5 人**,不可改(若用户要换人,转 `roundtable-skill`)
3. 检查 5 个 skill 是否已安装 (本会话已确认全在)
4. 向用户简短确认 (1 行): "将召集 [5 位] 讨论 {议题}, ~8-15 分钟"
5. 可选: `web_search` 拉取议题最新动态 (30 秒, 失败不阻塞)

### Phase 1: 并行观点采集 (5 个 sub-agent, ~3-5 分钟)

**关键设计**: 由于 5 位委员都是高认知密度人格, 且用户偏好"主会话直接做"(参见 `roundtable-skill` 模式 D), 本 skill **默认采用模式 D (主会话串行)**,仅在用户明确要求"并行加速"时切换为子代理并行。

**模式 D (默认) 的执行模板** (每位委员):

```markdown
## 【N】{委员名} ({skill-name} · {3-4 个核心标签})

### 核心框架判断 (150-200 字)
[用该委员最具代表性的 1-2 个框架分析议题]

### 第二框架判断 (150-200 字)
[该委员的另一个核心框架]

### 第三框架判断 (150-200 字)
[第三个核心框架或独特视角]

### 核心判断 (200 字)
[该委员的最终判断,带棱角,带个人风格]

### 危险/警告 (100 字)
[该委员认为最值得关注的风险/反直觉点]

### 金句 (1 句)
[用该人格的标志性表达方式,加粗斜体]
```

**并行模式 (用户明确要求时)**:
对每位委员执行 `delegate_task`, 注入从对应 skill 提取的 3-5 个核心框架摘要 + 代表性引语 (300-500 字), 保证人格一致性。详见 `roundtable-skill` Phase 1。

**独立性保护**: 5 位委员在 Phase 1 互不知晓彼此 (无论串行/并行), 由主会话在 Phase 2 汇总。

### Phase 2: Dialogue Master 主持碰撞 (主会话, ~3-5 分钟)

**Step 2.1: 观点碰撞矩阵** (5 × 4 表格)

| 焦点维度                        | 爱因斯坦 | 达·芬奇 | Vitalik | Naval | Balaji |
| --------------------------- | ---- | ---- | ------- | ----- | ------ |
| **时间尺度判断** (1 年/10 年/100 年) |      |      |         |       |        |
| **核心瓶颈/路径**                 |      |      |         |       |        |
| **最大风险/警告**                 |      |      |         |       |        |
| **哲学观/思维特征**                |      |      |         |       |        |

**Step 2.2: 3 段虚拟对话碰撞** (借鉴 `roundtable-skill` 模式)

- **交锋 1**: 定义层面的最大分歧 (哪位委员和谁的根本定义不同?)
- **交锋 2**: 路径层面的核心矛盾 (推荐路径谁 vs 谁?)
- **交锋 3**: 赢家/利益层面的最尖锐分歧
- 每段对话: 每人 80-150 字回应, 共 400-750 字/段
- 必须包含至少 1 次**直接反驳** + Dialogue Master 查理式底层逻辑追问

**Step 2.3: 破局提问** (Dialogue Master 僵局破局)
问所有委员: "如果 10 年后回头看今天的判断, 你最可能'错'在哪里?"
每人 80-100 字的自我反省式回应。

**Step 2.4: 决议性张力分析** (本 skill 强化, 与 roundtable 通用报告区分)
Dialogue Master 总结 5 位的**核心共识** (3-5 条) + **不可调和的元分歧** (1-3 条, 这些分歧本身就是"决议"的关键)。

## 执行规则

### 规则 1: 委员硬编码, 不可临时增删

- 本 skill 的核心价值是**预配置 5 人天团**——临时换人等于破坏"委员会"特性
- 如要换人/加人/减人, 改用 `roundtable-skill` 通用圆桌

### 规则 2: 议题必须"开创性 / 高深 / 跨界 / 重大战略"

- 不适用问题: 简单事实查询、日常执行、普通运营问题
- 适用问题: "X 行业 10 年后的格局是什么？"、"该不该 All-in Y 方向？"、"如何用 Z 框架重构我的业务？"、"AB 路径哪个更值得 10 年投入？"
- 边界情况: 简单问题**不**应该用本 skill, 浪费 5 位大师的算力

### 规则 3: 报告必须"决议导向"

- 与 `roundtable-skill` 通用分析报告的关键差异
- 必须包含: 核心决议、不可调和的元分歧、风险红线、分层行动建议
- 不允许只输出"5 位观点"而无"决议"

### 规则 4: 独立性保护

- 5 位委员在 Phase 1 互不知晓彼此 (无论串行/并行)
- Phase 2 由 Dialogue Master 在主会话汇总碰撞
- 避免从众 (一位委员的观点不能影响其他委员的初始判断)

### 规则 5: 容错机制

- 单个委员 sub-agent 超时 > 3 分钟: 跳过并说明
- 5 个 skill 中任一未安装: 提示用户安装命令
- `web_search` 失败: 不阻塞, 背景材料标"无"
- INDEX.md 写入失败: 不阻塞主报告保存

### 规则 6: 复用而非另起炉灶

- 4 阶段流程细节 = `roundtable-skill` Phase 1-4
- 主会话串行模式 D = `roundtable-skill` 模式 D
- 7 维风格指纹 = 各委员 skill 内部定义
- 本 skill 不重复实现这些, 仅做**硬编码 + 决议报告增强**

## 进阶模式

### 模式 E1: 追问模式 (报告完成后)

```
# 触发词
"请 {某委员} 深入展开他的 {某观点}"
```

针对该委员的观点, 再次用该人格深入分析。详见 `roundtable-skill` 模式 A。

### 模式 E2: 决议迭代模式 (多轮真实迭代)

```
# 触发词
"奇才委员会 (高级模式): {议题}"
```

- Phase 1: 5 位并行采集原始观点
- 追加轮次 1: 把 B 的观点喂给 A, 要求 A 反驳
- 追加轮次 2: 把所有前轮观点喂给所有委员, 各自再回应
- 终极轮: Dialogue Master 总结
- 代价: token 消耗 2-3 倍, 耗时 20-30 分钟

### 模式 E3: 时空穿越混搭

```
# 触发词
"奇才委员会 + 古代哲人: {议题}"
```

在 5 位常驻委员基础上, 临时加入 1-3 位古代哲人 (如 `dialogue-laozi` / `dialogue-kongzi`), 构成"古今混搭决议"。详见 `roundtable-skill` 模式 C。

### 模式 E4: 单委员深度模式

```
# 触发词
"用爱因斯坦的视角深度分析: {议题}"
```

绕过 5 人圆桌, 直接调用某个委员的 skill 深度分析。**注意**: 这等于绕过本 skill, 应直接用对应 skill (如 `einstein-method`)。

## Skill 局限性声明

1. **5 人硬编码的代价**: 议题若更适合其他人物 (如纯哲学议题更适合孔子/苏格拉底), 本 skill 不是最优选择——应改用 `roundtable-skill`。
2. **决议报告的"虚"风险**: 5 位 AI 模拟的人物不可能真正"决议", 用户应将报告作为**思考支架**, 而非终极答案。
3. **token 与耗时**: 默认模式 D (主会话串行) 5 位委员总耗时 8-15 分钟, 高级模式 20-30 分钟——不适用于"快速问个问题"的场景。
4. **议题门槛**: 不适用普通问题 (如 "我应该学 Python 还是 Go?")——浪费 5 位大师的算力, 简单问题直接搜索更高效。
5. **不替代实际决策**: AI 决议报告仅作为思维工具, 真正的重大决策应结合实际数据、专业咨询、用户自身价值观。
