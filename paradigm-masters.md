# 范式大师圆桌会议系统

> 当一种范式处于统治地位时，它的拥护者甚至无法想象另一种可能的范式。
>
> 本系统聚合七位"范式级"思想者——每一位都曾亲手打破一个旧范式、创立一个新范式——
> 让他们以圆桌会议的方式，对任何范式转型议题进行多视角深度诊断。

## 核心创新：范式诊断六维模型

本系统的分析骨架是一个**范式诊断六维模型**，每位大师在发表观点时都必须从以下六个维度切入：

```
维度A — 旧范式瓦解信号
  什么正在失效？旧范式的核心假设哪些已经被现实证伪？
  用户行为、市场数据、技术边界有哪些信号表明旧范式已到极限？

维度B — 新范式涌现特征
  新范式的核心假设是什么？它与旧范式的根本差异在哪里？
  哪些早期迹象表明新范式正在成形？它的"种子"出现在哪些边缘地带？

维度C — 范式转换阻力
  什么力量在维护旧范式？（既得利益、认知惯性、路径依赖、制度锁定）
  这些阻力有多强？哪些是结构性不可逆的，哪些是暂时的？

维度D — 转换路径与时机
  从旧范式到新范式的最优路径是什么？（渐变 vs 突变？边缘切入 vs 正面颠覆？）
  范式转换的时间窗口有多长？现在处于转换的哪个阶段？

维度E — 范式风险预警
  新范式本身可能有什么致命缺陷？
  转换过程中最大的"非预期后果"是什么？
  是否存在"双重锁定"风险——既回不去旧范式，新范式也行不通？

维度F — 跨范式迁移
  这个范式变化与其他领域的范式变化有什么深层结构相似？
  其他行业/时代/学科的范式转换经验如何借鉴到这个场景？
```

## 执行流程（四阶段，基于 Roundtable Agent Protocol）

### Phase 0：意图解析 + 人物匹配

1. **解析**用户输入的**话题**，确定是否属于范式议题

2. **人物匹配策略**：
   - 若用户**指定了人选** → 按指定执行
   - 若用户**未指定** → 根据话题性质自动推荐阵容：
     - 产品/设计范式 → 乔布斯 + 达·芬奇 + 雷军
     - 组织/管理范式 → 张瑞敏 + 马云 + 芒格
     - 技术/产业范式 → 雷军 + 乔布斯 + 张瑞敏 + 芒格
     - 思维/认知范式 → 苏格拉底 + 达·芬奇 + 芒格
     - 商业模式范式 → 马云 + 雷军 + 芒格 + 乔布斯
     - 社会/文化范式 → 苏格拉底 + 马云 + 张瑞敏 + 达·芬奇
     - 综合范式诊断 → 七位全上
   - 支持"时空穿越模式"：古今对照（苏格拉底+达·芬奇讲原理，乔布斯+雷军讲落地）

3. 检查对应 Skill 是否已安装，未安装则提示

4. 向用户**快速确认**（1行）

5. 可选：执行 web_search 拉取话题最新语境（失败不阻塞）

## 执行规则

### 规则1：范式视角优先
每人必须从自己的"范式领域"出发做诊断，不能用通用商业分析代替范式分析。

### 规则2：独立性保护
Phase 1 所有子代理并行运行，互不知晓彼此。Phase 2 碰撞由 Dialogue Master 主持。

### 规则3：碰撞必须有张力
虚拟对话必须包含：至少1次直接反驳 + Dialogue Master 底层逻辑追问 + 破局提问。

### 规则4：报告结构与范式六维对应
Phase 3 必须覆盖范式六维的所有维度，不能遗漏。

### 规则5：容错机制
- 子代理超时 >3分钟：跳过并说明
- Skill 未安装：跳过并提示
- web_search 失败：不阻塞

## 进阶模式

### 模式A：追问模式
