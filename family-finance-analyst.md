# 家庭财务分析专家（Family Finance Analyst）

> 基于金蝶随手记 APP 导出的家庭收支数据，构建从数据接入→清洗存储→多维度分析→财务建议→报告输出的完整流水线。
> 数据本地化处理，不上传第三方，不做自动支付/转账操作。

---

## 一、Skill 定位

### 1.1 核心能力

- **数据接入**：读取随手记 APP 导出的 CSV 文件，自动清洗与标准化
- **多维度分析**：核心指标、支出结构、趋势分析、异常检测
- **深度挖掘**：识别「拿铁因子」、分类校验、模式发现
- **智能建议**：基于基线对比和个性化目标，生成可执行的财务改进建议
- **报告输出**：生成结构化 Markdown 分析报告 + 可选 HTML 可视化报告

### 1.2 红线约束

| 红线 | 说明 |
|------|------|
| ❌ 数据不离开本地 | 所有财务数据在本地处理，不上传任何第三方服务 |
| ❌ 不做自动支付 | 不连接支付接口，不执行任何转账、扣款操作 |
| ✅ 只分析不存储 | 分析完成后不持久化存储完整交易明细，仅保留汇总指标和建议 |

## 二、数据接入流程

### 2.1 从随手记导出数据

用户需手动从随手记 APP 导出数据，步骤如下：

```
1. 打开随手记 APP → 侧边栏 → 「数据管理」
2. 选择「导出全部数据」
3. 格式选择 CSV（免费，通用性强）
4. 如有多个账本或需分批导出，可按账本/时间段导出
```

### 2.2 文件存放规范

导出的 CSV 文件建议按以下结构存放：

```
~/family-finance-data/
├── raw/                    # 原始导出文件（只读，不修改）
│   ├── 2024_Q1.csv
│   ├── 2024_Q2.csv
│   └── 2025_Q1.csv
├── processed/              # 清洗后的数据文件
│   └── household_finance.db  # SQLite 数据库
└── reports/                # 生成的报告
    ├── 2025_年度财务报告.md
    └── 2025_年度财务报告.html
```

### 2.3 数据清洗流程

加载本 skill 后，优先检查 `~/family-finance-data/` 目录是否存在最新数据：

```
1. 检查 raw/ 目录中是否有新的 CSV 文件（未处理过的）
2. 如有 → 调用 Python 脚本进行清洗和转换
3. 如无 → 使用现有 processed/ 数据进行分析
4. 如数据目录不存在 → 提示用户先导出随手记数据并创建目录
```

## 三、数据处理流水线

本 skill 附带一个 Python 数据处理脚本 (`scripts/process_data.py`)，用于将随手记 CSV 转换为标准化数据。

### 3.1 脚本功能

| 功能 | 说明 |
|------|------|
| CSV 读取 | 自动识别随手记 CSV 的列结构和编码（GBK/UTF-8） |
| 字段标准化 | 统一映射随手记的字段名 |
| 日期处理 | 标准化日期格式为 ISO 8601 |
| 缺失值处理 | 标记或填充缺失的分类、备注等字段 |
| 重复检查 | 基于交易日期+金额+分类+备注去重 |
| 分类映射 | 将随手记的子分类映射到一级分类，便于聚合分析 |
| 入库存储 | 将清洗后的数据导入 SQLite 数据库 |

### 3.2 激活方式

当用户提供 CSV 文件路径时，数据处理流程自动触发。

```bash
python <skill_dir>/scripts/process_data.py \
  --input <csv_path> \
  --output <db_path>
```

参数说明：

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `--input` | (必填) | 随手记导出的 CSV 文件路径，或目录路径（批量处理） |
| `--output` | `~/family-finance-data/processed/household_finance.db` | 输出 SQLite 数据库路径 |
| `--encoding` | 自动检测 | CSV 文件编码（常见：utf-8, gbk） |
| `--standardize` | true | 是否执行分类标准化映射 |

## 四、分析维度

### 4.1 核心指标计算

| 指标 | 计算方式 | 说明 |
|------|---------|------|
| 总收入 | 区间内所有收入类交易之和 | — |
| 总支出 | 区间内所有支出类交易之和 | — |
| 净储蓄 | 总收入 - 总支出 | 正数为储蓄，负数为透支 |
| 储蓄率 | 净储蓄 / 总收入 x 100% | 建议 >= 30% 为健康 |
| 日均支出 | 总支出 / 天数 | 用于日常预算参考 |
| 恩格尔系数 | 食品支出 / 总支出 x 100% | 衡量家庭富裕程度的参考指标 |

### 4.2 支出结构分析

对总支出按分类进行聚合分析：

| 分析项 | 图表类型 | 洞察方向 |
|--------|---------|---------|
| 类别占比 | 饼图 / 环形图 | 最大支出类别、各类别占比与参考基线对比 |
| 月度趋势 | 折线图 | 各项支出随月份变化趋势、季节性规律 |
| 同比/环比 | 对比柱状图 | 同一类别在不同时期的支出变化 |
| 子类明细 | 堆叠柱状图 | 一级分类下的二级分类构成 |

### 4.3 异常检测

通过统计方法自动标记异常：

```
检测方法：
- 单类别支出超过历史均值 +/- 2 倍标准差
- 某笔交易金额超过同类交易中位数 5 倍以上
- 某月总支出较前三个月均值波动超过 50%

输出：
- 高亮异常项，标注偏离幅度
- 询问用户是否有特殊原因（大额购物、医疗等）
```

### 4.4 「拿铁因子」识别

识别那些单笔金额小但频率高、累积起来很可观的非必要开支：

```
判定标准：
1. 单笔金额 < 50 元
2. 月发生频率 >= 5 次
3. 分类属于非必要消费（饮品、零食、外卖等）
4. 月度累计 >= 300 元

输出示例：
「每月咖啡支出累计 480 元，年化约 5,760 元。若改为自制咖啡，每年可节省约 4,000 元。」
```

### 4.5 分类校验

AI 审查交易记录，识别可能归类错误的条目：

```
校验逻辑：
- 交易描述含关键词但分类不匹配（如描述含「加油」但分类不是「交通」）
- 连续同类交易中混入不同分类的单笔
- 大额交易分类与历史同类交易不一致

输出：
- 列出疑似分类错误的交易
- 建议更正后的分类
- 询问用户确认
```

## 六、参考基线表

在进行支出结构对比时，使用以下公认参考基准：

| 支出类别 | 参考占比 | 说明 |
|---------|---------|------|
| 住房 | 25-30% | 含房贷/租金、物业、水电气 |
| 食品 | 10-15% | 含在家做饭、外出就餐 |
| 交通 | 8-12% | 含油费/公共交通/车辆维护 |
| 教育 | 5-15% | 视家庭阶段浮动（有子女更高） |
| 医疗 | 5-10% | 含保险、药品、诊疗 |
| 娱乐 | 5-10% | 含旅游、电影、运动等 |
| 储蓄 | >= 30% | 收入储蓄率健康线 |
| 其他 | 5-15% | 含通讯、服装、日用品等 |

> **注意**：以上为通用参考值，实际健康比例因城市、家庭结构、收入水平而异。
> 分析时应结合用户历史数据和家庭实际情况给出判断，而非机械套用。
