当用户提出问题时，按以下步骤执行：

## 步骤一：理解问题

提取问题核心要素：领域（社会/政治/经济/军事/组织管理/科技/文化）、主体（个人/组织/国家/文明）、分析深度。如不明确，问清楚。

## 步骤二：检索历史平行案例

从训练知识中检索，覆盖以下维度（每个至少1-2个案例）：

1. **中国政治史**（《资治通鉴》+ 二十五史）：制度、变法、党争、政权更替、边疆民族
2. **社会经济史**（《食货志》+ 中国通史）：土地、货币、财政、社会结构
3. **军事与外交**（《资治通鉴》+ 世界通史）：战略、和战、制度、著名战役
4. **世界史平行**（世界通史）：同期对照、跨文明比较
5. **思想文化**：主流思想变迁、外来文化影响

## 步骤三：撰写研究报告

按此结构输出：

```
# [主题] — 以史为鉴研究报告

## 一、执行摘要
核心问题 + 选中的平行案例（2-5个）+ 核心启示（3-5条）

## 二、问题解析
问题本质、核心矛盾、关键变量

## 三、历史平行案例
每个案例格式：
### 案例X：[事件/人物]
- **来源**：《资治通鉴》卷XX / 《XX史》
- **场景还原**：背景 + 关键人物 + 动机
- **因果链条**：前因 → 经过 → 后果（短期+长期）
- **得失评价**：同时代 + 后世评价
- **与当下的异同**：结构相似点 + 关键不同点

## 四、深度比较
- 相似性分析（场景、动力、约束）
- 差异性分析（时代、技术、制度）
- 历史规律提取（规律 + 适用边界）

## 五、以史为鉴
- 对当下的启示（2-3条）
- 警示与风险（历史上的失败教训）
- 历史能动性（人可以做不同选择）

## 六、结论
核心发现 + 最重要的提醒 + AI局限声明

## 参考文献
每条史料标注出处
```

## 质量要求

1. 每个历史论断标注具体史书卷次出处
2. 因果链完整：前因→经过→后果
3. 同时分析相似点 AND 不同点
4. 不确定的史实标注"存疑"，绝不编造
5. 禁止使用表格，全部用段落或列表
6. 案例 ≥ 2个（概览）/ ≥ 4个（标准）/ ≥ 6个（全面）
7. 标题层级 ≥ 4级

## 保存

报告保存到 `~/.claude/skills/history-mirror/reports/`，命名格式：`history-mirror-report-<主题>-YYYY-MM-DD.md`。

生成后告知用户：报告路径、案例数量、史料来源范围、核心结论摘要。
