# Poem Committee Skill — 古诗词委员会圆桌对话系统

## 核心定位

一键召唤李白、李煜、李清照、辛弃疾、纳兰性德五位诗词大师，就指定话题并行讨论，生成综合鉴赏与创作报告，保存至 `~/文档/poem-committee/`。

## 三阶段执行流程

### Phase 0：话题解析与准备（主会话，约30秒）

1. **解析**用户输入的诗词话题（可以是某首具体诗词、某种题材、某种情感、某个诗人等）
2. 确认五位诗人全部在线（各自根据自身时代背景和风格作答）
3. 若 topic 为具体诗词名，执行 `web_search` 快速确认原文和背景（失败不阻塞）
4. 输出准备确认（1行）：
   
   ```
   📜 诗会已开，议题：「{话题}」，五位大师已在座。
   ```

### Phase 1：并行观点采集（Sub-agents，五人并行，每人独立，约1-2分钟）

对每位诗人执行 `sessions_spawn(mode="run", context="isolated")`：

```
你现在扮演 **{诗人名}({诗人简介})**，请以你的风格发言。

## 你的诗词风格特征
{诗人核心风格描述 + 代表性作品名句，200字以内}

## 今日议题
**「{用户话题}」**

## 背景参考（如有）
{web_search 结果摘要，不超过200字}

## 请依次完成：

### 一、我对此议题的核心判断（150字）
（用你自己的风格语言）

### 二、结合你的代表作谈此议题（150字）
（引用你自己的一句诗词，说明它与此议题的关联）

### 三、最值得关注/最危险的地方（100字）
（从创作或鉴赏角度指出）

### 四、金句
（用你自己的风格说一句总结性的话，至少包含你的标志性意象）

风格要求：全程使用{诗人名}的语气、措辞和思维方式，仿佛你真的在参加一个跨越千年的诗会。
字数：600-800字。
```

**注意**：五位诗人**互不知晓彼此**，各自独立作答，保证观点原创性。

所有子代理并行 spawn 后执行 `sessions_yield`，等待全部完成。

**Step 2.2：三段虚拟诗会对话**

> 背景：五位大师围坐一堂，就「{用户话题}」展开讨论。Dialogue Master 主持。

**交锋一：艺术手法之争**

- 议题：不同诗人对同一题材的艺术手法分歧
- 每位诗人60-100字回应
- 必须包含：至少2次直接反驳

**交锋二：情感深度之争**

- 议题：何种情感更具打动力
- 每人60-100字
- 纳兰性德与李清照的闺情 vs 李白辛弃疾的豪情

**交锋三：千古视角**

- 议题：这首作品/这个题材，千年后是否仍有共鸣？
- 每人50-80字

**破局提问**：
"若千年后有人读你的词，是否仍能感同身受？"
每位诗人50字以内的反思式回应。

## 执行规则

### 规则1：诗人人格一致性优先

子代理 task prompt 必须包含该诗人的核心风格特征（意象、语言习惯、情感基线），不可泛泛而谈。

### 规则2：独立性保护

Phase 1 五位诗人**互不知晓彼此**，各自独立作答。Phase 2 的碰撞由主持人在主会话内基于已采集成果进行。

### 规则3：碰撞必须有张力

虚拟对话不是"总结发言"，必须包含直接反驳和观点交锋。

### 规则4：报告有结构

报告必须包含：执行摘要、独立观点、碰撞矩阵、虚拟对话、共识与分歧、金句、创作建议；缺一不可。

### 规则5：容错机制

- 子代理超时 >3 分钟：跳过该诗人，在报告中注明"[李白超时未响应]"
- web_search 失败：背景材料标为"无"，不阻塞执行
