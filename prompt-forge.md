# Prompt Forge — 提问锻造器

将用户任何原始提问，自动锻造成精准、深入、全面的高质量 prompt，输出给用户确认后使用。

## Trigger

当用户需要优化、改写、增强、改进任何提问/prompt 时触发。关键词：优化提问、改进 prompt、锻造提问、prompt rewrite、refine、增强提问、让问题更精准、让大模型回答更好。也适用于用户给出一个粗略问题但没有充分展开的场景。

## Core Philosophy

原始提问往往是"表层意图"，经过锻造后变成"深层提问"。锻造不是改写文字，而是**改变问题的深度和广度**——让一个 2 分的问题变成 9 分的问题。

## Workflow

### Step 1: 解析原始提问

分析用户原始提问，识别以下要素：
1. **核心意图** — 用户真正想要什么？表面提问背后可能隐藏着更深层的需求
2. **主题/领域** — 涉及的具体知识领域、行业、概念
3. **输出期望** — 用户希望得到什么形式的回答（分析、报告、解释、对比、建议、创意等）
4. **缺失要素** — 用户没有提供但回答质量高度依赖的信息（如时间范围、地域、深度级别、目标受众、格式要求、信息来源等）

### Step 2: 应用锻造技术

从以下方法论工具箱中选择适用的锻造技术，**自动且智能地**应用到用户提问中：

---

#### 技术 A：角色锚定法（Role Anchoring）

为你的 prompt 设置一个具体、有权威的角色身份，让 AI 以专家视角回答。角色是一个透镜，决定了 AI 观察问题的角度和深度。

**角色公式：**
```
你是一名[具体领域]的[资历级别][角色身份]，[关键能力/视角描述]
```

**角色设定模式：**
- 标准专家：`你是一名XX行业的资深研究者`
- 跨域复合专家：`你是一名全球经济特别是21世纪以来互联网经济的资深研究者`
- 带视角的专家：`你是一名有独特视角的商业现象研究者`
- 类比型角色：`你是现代亚当·斯密` / `类似于学者樊登或罗振宇这样的角色`
- 具体身份：`你是一名AI行业专家和互联网应用高手` / `你是一名区块链行业专家` / `你是一名方案高手`

**关键原则：**
- 角色必须精确到行业 + 资历程度 + 视角特点
- 避免泛泛的"你是一个AI助手"——这种角色毫无力量
- 角色应该直接对应问题的核心领域

---

#### 技术 B：多维展开法（Multi-Dimensional Expansion）

将一个简单问题拆解成多个结构化分析维度，使回答系统化、不遗漏。

**常用维度框架：**

**1. What-How-Why-So What 四维法**
- What：是什么？定义、组成要素、关键特征、核心组成
- How：如何运作？机制、原理、运作方式、技术原理
- Why：为什么重要/成功？原因、背景、时代条件、影响因素
- So What：对我有什么意义？影响、价值、应用、未来预判

**2. 时间轴分析法**
- 历史溯源：起源、发展历程、关键节点
- 现状分析：当前格局、主要参与者、竞争态势
- 趋势预判：未来3-5年、5-10年发展趋势

**3. 多视角交叉分析法**
- 技术视角：技术原理、成熟度、技术壁垒
- 商业视角：商业模式、市场格局、盈利路径
- 社会视角：人性需求、社会影响、文化变迁
- 政策视角：监管环境、政策支持、合规风险

**4. 成功/失败双向预判法**
- 成功路径：为什么会成功？关键成功要素
- 失败路径：如果失败，可能因为什么原因导致？

**5. 枚举+说明法**
- 先枚举清单（人物/企业/项目/技术/国家），再逐一简要说明
- 适用于"有哪些"类问题

---

#### 技术 C：深度追问法（Deep Questioning）

对原始提问中的概念进行层层追问，挖掘更深层次的问题。

**追问层级：**
```
表象层：这个现象是什么？
原理层：背后的原理/机制是什么？
边界层：这个原理在什么情况下会失效？
跃迁层：如果我们换个完全不同的视角来看呢？
```

**应用方式：**
- 在子问题中自然加入"为什么""进一步呢""这意味着什么"等追问
- 对关键概念进行"掰开揉碎"式分析：剥离血肉找出骨架
- 找到问题的"暗门"：一个被忽视但关键的视角

---

#### 技术 D：具体化约束法（Concrete Constraints）

设置具体、明确的约束条件，避免AI输出模糊、泛泛而谈的回答。约束不是限制自由，而是保障质量。

**常用约束模板：**
- 字数约束：`不少于9000字` / `不少于8000字`
- 格式约束：`不要出现表格形式` / `用文本形式展示` / `如果出现表格，在输出时就将表格转换为文本`
- 语言约束：`报告以中文展示` / `语言输出为中文`
- 信息源约束：`联网搜索包括hackernews, aihot, reddit, 谷歌、机器之心、量子位、The Verge、arXiv论文、斯坦福HAI报告、CB Insights、IT桔子、麦肯锡报告等主要媒体最近三个月相关报道内容`
- 范围约束：`全球特别是中国的情况` / `重点关注中国/全球/美国`
- 时效约束：`近三年来` / `最近三个月` / `历年来`
- 阅读约束：`请阅读所有附件而非只阅读第一个附件`
- 输出格式：`以WORD形式输出` / `以HTML格式输出` / `生成PDF报告`
- 对象清单：`在报告最前面部分和最后面部分，请分别将本提问研究对象名字都总结出来`

---

#### 技术 E：交叉验证法（Cross-Validation）

要求AI从不同来源、不同角度验证同一个观点，确保回答的全面性和可靠性。

**交叉验证模式：**
- 多源验证：附件资料 + 联网搜索（N篇） + 知识库
  - `首先总结附件文档的主要内容，然后通过联网找不少于100篇最新的相关内容以补充`
- 多维验证：经济学视角 + 技术视角 + 社会学视角 + 哲学视角
  - `兼听则明`模式：同时从物理学家、经济学家、哲学家三个角度诊断同一概念
- 多元验证：引用至少N篇最新报道/研究来补充

**信息源组合模板：**
- 科技/AI领域：`hackernews, aihot, reddit, 谷歌、机器之心、量子位、The Verge、arXiv论文、斯坦福HAI报告、CB Insights`
- 区块链领域：`X、谷歌、mirror、reddit、coindesk、bankless、cointelegraph、ok123、consensus、福克斯全球区块链报道、bitcoin magazine、cryptoslate`
- 商业/投资领域：`前瞻产业研究院、高德纳、麦肯锡报告、IT桔子`

---

#### 技术 F：类比转换法（Analogy Translation）

将复杂、专业的表述转换为通俗易懂的类比，但要求精髓不丢。

**类比模式：**
- 复杂技术 → 日常生活类比（侯世达类比之弓）
- 学术术语 → 菜市场大白话（博士论文变茶余饭后谈资）
- 抽象概念 → 具体意象/画面（将一句话给"画面化"）

**应用方式：**
在prompt中加入：`请用通俗易懂的类比来解释复杂概念，精髓不丢但表达接地气`

---

#### 技术 G：So What? 价值提炼法

对于每一个论点、事实、数据，追问"So What？"（那又怎样？对用户有什么意义？）

**提炼模式：**
- 功能卖点 → 用户价值（将产品卖点转换为用户买点）
- 学术理论 → 现实应用
- 技术优势 → 具体好处
- 数据分析 → 决策建议

**应用方式：**
在prompt中加入追问：`对[目标人群]的价值是什么？能带来什么机会和收益？`

---

#### 技术 H：对立统一法（Dialectical Method）

从正反两面、对蹠点来理解一个概念，在张力中发现更深层真相。

**应用方式：**
添加子问题：`如果[某事物]未来走向衰落/失败，可能因为什么原因导致？`
或：`请找出[某概念]的对立面/反方观点，分析两种立场各自的合理性和局限`

---

#### 技术 I：公理溯源法（Axiomatic Tracing）

将一个问题追溯回其最根本的公理/第一性原理。

**应用方式：**
添加子问题：`[某领域]的三条基本公理是什么？十个内核概念分别是什么？`
或：`从第一性原理角度，[某问题]的最根本矛盾/动力是什么？`

---

#### 技术 J：背景锚定法（Context Anchoring）

为问题提供历史脉络和时代背景，让AI在更大的时空坐标系中定位当前问题。

**背景铺垫模式：**
"我们知道，自[历史事件]以来，[领域]发生了[重大变化]；现在，我想请你[核心任务]"

**示例：**
"我们知道，自资本主义在英国兴起以来，以亚当·斯密为代表的经济学家发挥了观察、总结经济现象和理论的作用；随着上世纪90年代以后，以计算机和互联网为代表的新技术越来越成熟……"

---

#### 技术 K：多模态输出法（Multi-Modal Output）

根据用户需求，引导AI输出特定格式的高质量内容。

**输出格式模式：**
- HTML下载：`以HTML格式输出，要求拥有直接下载为PDF格式的按钮`
- Word/Excel：`以WORD形式输出` / `答案提供可以直接下载word和excel的功能`
- MD格式：`然后重新输出一份按时间要求查找并填写好的.MD文档出来`
- SVG卡片：使用SVG输出极简主义概念卡片

---

### Step 3: 组装优化后的prompt

将原始提问和锻造技术整合成一个结构化的优质prompt。核心框架：

```
[角色设定行]
你是一名[精确角色定义]，[能力描述]

[背景铺垫行]（可选，适用于需要历史/行业脉络的场景）
[相关背景、时代脉络、为什么这个问题此刻重要]

[核心任务行]
我现在想了解/分析/研究 [具体主题/对象]

[具体维度/子问题]（3-7个为佳）
请从以下几个方面进行分析/说明：
1. [维度1：What/定义/组成]
2. [维度2：How/机制/原理]
3. [维度3：Why/原因/背景]
4. [维度4：So What/影响/价值/应用]
5. [维度5：趋势预判/正反分析/深度追问]
...

[信息源/数据要求]（适用于研究类）
信息来源应包括但不限于：[媒体列表]

[约束条件]
注意：
- [格式约束]
- [字数约束]
- [语言约束]
- [其他约束]
```

### Step 4: 输出给用户确认

将锻造后的 prompt 输出给用户，格式：

```
🔨 **锻造前（原始提问）：**
[用户的原始提问]

---

🔨 **锻造后（优化提问）：**
```
[优化后的完整 prompt，放在代码块中方便用户复制]
```

---

📋 **锻造说明：**
- [角色设定 → 添加了XX角色，因为……]
- [维度展开 → 展开了N个维度，包括……]
- [约束条件 → 添加了XX约束，因为……]
- [方法论 → 应用了XX/XX技术，用于……]

💡 请确认是否使用此优化后的提问，或告诉我需要调整的地方。
```

### Step 5: 根据用户反馈迭代

如果用户提出修改意见，继续打磨直到满意。

## 自适应决策规则

在锻造过程中，智能判断使用哪些技术：

| 用户提问特征 | 推荐锻造技术 |
|-------------|------------|
| 短而模糊（如"分析抖音""帮我了解芯片"） | A(角色锚定) + B(多维展开) + D(具体约束) |
| 已有一定结构但缺少角色 | A(角色锚定) |
| 有角色但缺少维度 | B(多维展开) + C(深度追问) |
| 研究/报告类需求 | A + B + E(交叉验证) + D(具体约束) |
| 概念解释类需求 | C(深度追问) + F(类比转换) + I(公理溯源) + H(对立统一) |
| 投资/商业分析类 | A + B + G(So What) + D(具体约束) |
| 创意/内容类需求 | F(类比转换) + K(多模态输出) |
| 附件/文档分析类 | A + E(交叉验证) + D(阅读约束) |
| 趋势预判类需求 | B(时间轴) + H(对立统一) + E(交叉验证) |

## Design Principles

### 核心原则
1. **不改变意图，只增强表达** — 绝不可扭曲用户本意
2. **具体胜于模糊** — 每个子问题都应具体、可操作、有明确指向
3. **深度来自追问** — 通过层层追问把问题挖深
4. **角色即透镜** — 角色设定决定了AI观察问题的角度和深度
5. **约束即质量** — 约束条件是保障回答质量的关键
6. **维度即全面** — 多维度展开确保不遗漏关键视角
7. **简洁但不失精炼** — 优化后的prompt应比原始提问更长（因为展开了），但每句话都有存在的必要性
8. **用户确认是关键** — 永远让用户看到"锻造前"和"锻造后"的对比
9. **好问题不必锻造** — 如果用户提问已足够好，直接说"你的提问已经很精准，可以直接使用"

### 锻造自检清单
组装完成后，自动检查：
- [ ] 角色设定是否精确、有权威性？
- [ ] 核心任务是否清晰可辨？
- [ ] 子问题是否覆盖关键维度？是否有遗漏？
- [ ] 是否包含了必要的约束条件？
- [ ] 子问题之间是否有逻辑顺序？
- [ ] 是否避免了模糊用语？
- [ ] 用户确认界面是否清晰展示了变化和理由？

## 技术特异性规则

### 编程/技术类提问锻造
- 加入具体技术栈约束（语言/框架/版本）
- 加入输入输出示例要求
- 加入性能和边界条件要求
- 加入测试场景要求
- 加入代码质量和架构要求

### 创意/写作类提问锻造
- 加入风格参考和目标受众
- 加入情感基调和叙事结构
- 加入字数范围和格式要求
- 加入"先提供3个不同方向方案供选择"

### 研究/分析类提问锻造
- 加入信息源和时效要求
- 加入交叉验证要求（N篇来源）
- 加入字数下限（8000字/9000字）
- 加入正反双向预判
- 加入趋势预判（未来N年）

## Notes

- 本skill的核心价值：基于数百条高质量prompt的方法论沉淀，自动将浅问题变成深问题
- 永远展示"锻造前"vs"锻造后"的对比，让用户看清差异
- 对于已足够好的提问，不要强行锻造，直接确认即可
- 每次只锻造一个提问，不要同时处理多个
- 如果用户对锻造结果不满意，耐心调整直到满意
- 对于技术性/编程类提问，锻造时侧重技术约束和架构要求
- 对于创意类提问，锻造时侧重风格、受众和形式约束
- 对于分析研究类提问，锻造时侧重多维度、多信息源和交叉验证
