# Agentic Research & Report Generation

## 概述

本 skill 复刻 [WiseMind AI](https://wisemindai.app/) 的核心功能：一个由 AI Agent 驱动的深度联网研究与报告生成平台。通过系统化的多源搜索、信息提取、交叉验证和结构化写作，产出高质量、含引用来源的深度研究报告。

### WiseMind AI 核心功能清单

WiseMind AI 提供以下核心能力，本 skill 逐一实现：

- **深度联网研究** — 基于研究课题执行多轮多维度搜索，覆盖学术、行业、新闻、社交媒体、链上数据等多源信息
- **多源信息交叉验证** — 每个关键数据至少2个独立来源验证，标注信息可信度
- **智能框架构建** — 自动生成4+级标题结构的报告大纲，为每个章节分配信息素材
- **结构化报告撰写** — 中文输出，含执行摘要、核心发现、深度分析、趋势研判、风险评估、战略建议、参考文献
- **引用格式与来源追踪** — 正文[来源1]标注+文末参考文献列表含URL
- **多格式导出** — Markdown (.md)、Word (.docx)、PDF (.pdf)

## 工作流程

### 阶段一：接收研究课题并解析需求

1. 从用户提问中提取：
   
   - **研究主题**：核心关键词或课题名称
   - **研究类型**：选择以下类型之一
     - 学术论文综述 (academic-review)
     - 行业研究报告 (industry-research)
     - 技术分析报告 (tech-analysis)
     - 竞品分析报告 (competitive-analysis)
     - 政策分析报告 (policy-analysis)
     - 综合深度报告 (comprehensive)
   - **时间范围**：默认近12-24个月，用户可指定
   - **深度级别**：概览型(3000-5000字) / 标准型(9000-15000字) / 全面型(20000字+)
   - **输出格式**：默认 Markdown (.md)，可选 docx/pdf

2. 若信息不全，做出合理假设并向用户说明

3. 加载对应的报告模板（详见 `references/report-templates.md`）

### 阶段二：多维度深度搜索

对研究课题执行至少 **8轮**、覆盖不少于 **4个维度** 的搜索：

#### 维度一：学术与科研信息源（≥2轮）

- `site:scholar.google.com <关键词>` | `site:arxiv.org <关键词>`
- `site:nature.com <关键词>` | `site:ieee.org <关键词>`
- `site:ssrn.com <关键词>` | `site:sciencedirect.com <关键词>`

#### 维度二：行业与市场数据（≥2轮）

- `"<关键词>" market size forecast 2025 2026`
- `"<关键词>" industry report analysis trends`
- `"<关键词>" revenue growth adoption rate statistics`

#### 维度三：新闻与媒体评论（≥2轮）

- `"<关键词>" latest news 2026`
- `site:cointelegraph.com <关键词>` | `site:coindesk.com <关键词>`
- `site:techcrunch.com <关键词>` | `site:reuters.com <关键词>`

#### 维度四：技术与项目动态（≥2轮）

- `site:github.com <关键词>` | `site:docs.<project>.org`
- `site:mirror.xyz <关键词>` | `site:reddit.com <关键词>`
- `"<关键词>" roadmap update partnership funding launch`

#### 搜索执行规则

1. 每维度至少2轮搜索
2. 搜索结果去重与信息增量筛选
3. 标注每个信息源的类型（学术/新闻/官方/社区）
4. 优先使用近12个月信息
5. 关键数据（市场规模、用户数、融资额）必须至少2个信息源交叉验证
6. 每轮搜索记录到工作日志，便于追溯

### 阶段三：信息整理与框架构建

1. **去重与质量筛选**
   
   - 合并重复信息，保留最完整/最新版本
   - 标注信息可信度：高（官方/权威媒体）、中（行业媒体/社区）、低（博客/社交媒体）

2. **构建报告大纲**（不少于4级标题）
   根据研究类型，使用对应模板生成大纲结构：
   
   ```
   一、执行摘要
   二、研究背景与方法
   三、核心发现
     3.1 子主题A
       3.1.1 细分点A-1
         3.1.1.1 详细论据1
         3.1.1.2 详细论据2
       3.1.2 细分点A-2
     3.2 子主题B
   四、深度分析
     4.1 ...
   五、趋势研判
   六、风险评估
   七、战略建议
   八、结论
   附录：参考文献
   ```

3. **为每个章节分配信息素材**
   
   - 将搜索结果归类到对应章节
   - 确保每个子章节至少有2个独立信息源支撑

### 阶段四：撰写研究报告

#### 写作质量要求

1. **每个论点/数据标注来源**：正文中使用 `[来源X]` 引用标记
2. **禁止使用表格**：所有对比用段落或项目符号列表呈现
3. **中文输出**：报告正文使用中文，项目名称/专业术语保留英文原名
4. **数据时效性**：关键数据标注时间戳（如：2026年5月数据）
5. **交叉验证**：关键数据必须有至少2个来源

#### 引用格式（详见 `references/citation-formats.md`）

- 正文引用：`[来源1]`、`[来源2][来源3]`

- 参考文献条目：
  
  ```
  [1] 作者/机构. "文章标题." 来源媒体, 发布日期. URL. 访问日期.
  [2] ...
  ```

### 阶段五：报告审核与导出

1. **质量检查清单**
   
   - [ ] 字数达到用户要求
   - [ ] 无表格（全文检查，不含任何 Markdown 表格）
   - [ ] 所有关键数据有来源标注
   - [ ] 参考文献列表完整，含URL
   - [ ] 标题层级 ≥ 4 级
   - [ ] 信息源不少于 20 个
   - [ ] 执行摘要准确概括核心发现

2. **导出文件**
   
   - Markdown：保存为 `wisemind-report-<主题简述>-YYYY-MM-DD.md`
   - Word：使用 `libreoffice --headless --convert-to docx <file>` 转换
   - PDF：使用 `libreoffice --headless --convert-to pdf <file>` 转换

## AI 辅助分析框架

在研究报告撰写过程中，应用以下分析框架确保深度：

### SWOT 分析

对关键项目/赛道进行优势(Strengths)、劣势(Weaknesses)、机会(Opportunities)、威胁(Threats)四维分析。所有维度用项目符号呈现，不用表格。

### PEST 分析

从政治(Political)、经济(Economic)、社会(Social)、技术(Technological)四个宏观维度评估外部环境。

### 波特五力模型

用于行业竞争格局分析：新进入者威胁、替代品威胁、买方议价能力、卖方议价能力、现有竞争强度。

### 价值链分析

梳理行业价值链各环节，识别价值创造的核心节点和价值分配格局。

## 注意事项

- **绝不编造数据或引用**：无法确认的信息明确标注"信息缺失"或"待验证"
- **搜索时效性**：结果需标注时间，优先12个月内最新信息
- **客观中立**：对每个分析结论保持客观，标明信息置信度
- **信息缺口**：无法从公开渠道获取的信息明确说明而非推测
- **格式遵循**：遵循用户的格式偏好和行业术语习惯
- **禁止表格**：全文不使用任何 Markdown 表格（引用格式规范除外）
